## Summary

Investigate and optionally support dispatching selected workflows to Claude Code so the project can take advantage of Anthropic subscription plans where that makes economic or operational sense.

## Motivation

Some of the current planning, execution, and review flows use Anthropic models through normal model routing. If Claude Code can be used as an execution backend for certain subagent-style tasks, it may provide better cost characteristics for users already paying for Anthropic subscription plans.

## Candidate workflow pairs

### 1. Plan generation / plan review
- `generate-plan` planner dispatch
- `plan-reviewer` dispatch

### 2. Plan execution / code review
- `execute-plan` worker execution
- `requesting-code-review` / `refine-code` review dispatch

## Questions to answer

1. Can Claude Code be used as a dispatch target from this repo's orchestration model?
2. If yes, which task types are a good fit?
   - planning
   - code implementation
   - code review
   - review/remediation loops
3. What are the capability and tooling differences versus current pi subagent dispatch?
4. What context, filesystem, and git constraints would apply?
5. How should routing be configured?
   - explicit user option
   - per-skill setting
   - model tier mapping
   - fallback behavior
6. What are the cost, latency, and reliability tradeoffs?

## Expected outcome

Produce a recommendation and, if viable, an implementation approach for optional Claude Code routing in some or all of these flows.
