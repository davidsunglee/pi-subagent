## Summary

Replace the current subagent extension (nicobailon/pi-subagents) with the reference implementation from badlogic/pi-mono. The reference implementation is a thinner dispatch layer that aligns better with this repo's skill-as-orchestrator pattern, where SKILL.md files drive workflow and the subagent tool is a dispatch mechanism.

## Motivation

The current extension (Nico's) is a full orchestration framework (~30+ source files) whose chain-centric design competes with the orchestration already built into generate-plan and execute-plan skills. Mario's reference implementation (~1100 lines) provides the core single/parallel/chain dispatch without imposing its own workflow, and is maintained by the pi author for guaranteed API compliance.

See `.pi/subagent_analysis.md` for the full comparison.

## Migration steps

### 1. Install the reference implementation
- Source: https://github.com/badlogic/pi-mono/tree/main/packages/coding-agent/examples/extensions/subagent
- Replace the current extension files

### 2. Port custom agents
- Migrate agent definitions from `agent/agents/` to be compatible with the new discovery paths (`~/.pi/agent/agents/` or `.pi/agents/`)
- Verify frontmatter fields are compatible (the reference implementation uses a subset of Nico's fields)
- Agents to port: planner, coder, plan-reviewer, code-reviewer, code-refiner

### 3. Add per-task model override in parallel mode
- Add `model?: string` to the parallel tasks schema in the tool definition
- Pass the model override through to `runSingleAgent()`
- This is required by execute-plan Step 7 which dispatches wave tasks with resolved model tiers
- Estimated scope: ~10 lines

### 4. Consider adding model fallback
- Cherry-pick the ordered-retry pattern from Nico's `model-fallback.ts`
- Try primary model, catch retryable errors (rate limit, auth, overloaded), try fallback list
- Estimated scope: ~50 lines
- This is optional but improves reliability

### 5. Verify skill compatibility
- Run generate-plan end-to-end: single planner dispatch, plan-reviewer dispatch, edit loop
- Run execute-plan end-to-end: wave parallel dispatch, status code parsing, retry logic
- Confirm agent discovery finds agents from the expected paths
- Confirm model override works in parallel dispatch

### 6. Remove Nico's extension
- Remove all files from the old extension
- Clean up any extension-specific config at `~/.pi/agent/extensions/subagent/config.json`

## Features intentionally dropped

These Nico's features are not needed because the skills already handle them:
- **Chain orchestration** - skills own the workflow sequence
- **Intercom bridge** - skills handle inter-step communication
- **Fork context** - fresh context is always used
- **Async/background execution** - skills wait for results synchronously
- **Agent CRUD TUI** - agents are defined in the repo
- **Recursion depth guard** - subagents don't spawn subagents in this model
- **Chain directory** - artifacts live in `.pi/plans/`, not ephemeral temp dirs
- **Skill injection** - agent prompts are assembled by the skill via prompt templates

## Completion criteria

- Reference implementation is installed and functional
- Per-task model override works in parallel mode
- All existing agents are discovered and dispatched correctly
- generate-plan skill completes end-to-end (plan + review loop)
- execute-plan skill completes end-to-end (wave dispatch + verification)
- Old extension is fully removed
